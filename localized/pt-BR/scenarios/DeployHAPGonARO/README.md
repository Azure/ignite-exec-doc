## Verificar se existem pré-requisitos

A seguir, verifique se existem pré-requisitos. Esta seção verifica a existência dos seguintes pré-requisitos: RedHat OpenShift e kubectl.

## Criar um grupo de recursos

Um grupo de recursos é um contêiner para recursos relacionados. Todos os recursos devem ser colocados em um grupo de recursos. Criaremos um para esse tutorial. O comando a seguir cria um grupo de recursos com os parâmetros $RG_NAME, $LOCATION e $RGTAGS definidos anteriormente.

```bash
export RGTAGS="owner=ARO Demo"
export LOCATION="westus"
export LOCAL_NAME="arodemo"
export SUFFIX=$(tr -dc A-Za-z0-9 </dev/urandom | head -c 6; echo)
export RG_NAME="rg-${LOCAL_NAME}-${SUFFIX}"
az group create -n $RG_NAME -l $LOCATION --tags $RGTAGS
```

## Criar VNet

Nesta seção, você criará uma Rede Virtual (VNet) no Azure. Comece definindo diversas variáveis de ambiente. Essas variáveis conterão os nomes da sua VNet e das suas sub-redes, além do bloco de CIDR para a sua VNet. Em seguida, crie a VNet com o nome e o bloco de CIDR especificados em seu grupo de recursos usando o comando az network vnet create. Esse processo pode levar alguns minutos.
    
```bash
export VNET_NAME="vnet-${LOCAL_NAME}-${SUFFIX}"
export SUBNET1_NAME="sn-main-${SUFFIX}"
export SUBNET2_NAME="sn-worker-${SUFFIX}"
export VNET_CIDR="10.0.0.0/22"
az network vnet create -g $RG_NAME -n $VNET_NAME --address-prefixes $VNET_CIDR
```

## Criar uma entidade de serviço para o cluster do ARO

Nesta seção, você criará uma entidade de serviço para seu cluster do Red Hat OpenShift no Azure (ARO). A variável SP_NAME conterá o nome da sua entidade de serviço. A variável SUBSCRIPTION_ID, que pode ser recuperada usando o comando az account show, armazenará sua ID de assinatura do Azure. Essa ID é necessária para atribuir funções à sua entidade de serviço. Depois disso, crie a entidade de serviço usando o comando az ad sp create-for-rbac. Para terminar, extraia a ID e o segredo da entidade de serviço da variável servicePrincipalInfo, cujo resultado é um objeto JSON. Esses valores serão usados mais tarde para autenticar seu cluster do ARO no Azure.

```bash
export SP_NAME="sp-aro-${LOCAL_NAME}-${SUFFIX}"
export SUBSCRIPTION_ID=$(az account show --query id --output tsv)
servicePrincipalInfo=$(az ad sp create-for-rbac -n $SP_NAME --role Contributor --scopes /subscriptions/$SUBSCRIPTION_ID/resourceGroups/$RG_NAME --output json)
SP_ID=$(echo $servicePrincipalInfo | jq -r '.appId')
SP_SECRET=$(echo $servicePrincipalInfo | jq -r '.password')
```

## Implantar o cluster ARO

Nesta seção, você implantará um cluster do Red Hat OpenShift do Azure (ARO). A variável ARO_CLUSTER_NAME conterá o nome do seu cluster do ARO. O comando az aro create implantará o cluster do ARO com o nome, o grupo de recursos, a rede virtual, as sub-redes e a entidade de serviço especificados. Esse processo poderá levar cerca de 30 minutos para ser concluído.
    
```bash
export ARO_CLUSTER_NAME="aro-${LOCAL_NAME}-${SUFFIX}"
echo ${YELLOW} "This will take about 30 minutes to complete..." 
az aro create -g $RG_NAME -n $ARO_CLUSTER_NAME --vnet $VNET_NAME --master-subnet $SUBNET1_NAME --worker-subnet $SUBNET2_NAME --tags $RGTAGS --client-id ${SP_ID} --client-secret ${SP_SECRET}
```
