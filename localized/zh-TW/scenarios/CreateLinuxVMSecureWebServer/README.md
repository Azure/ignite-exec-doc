# 建立透過 HTTPS 保護的 NGINX Web 服務器

若要保護網頁伺服器，傳輸層安全性（TLS），先前稱為安全通訊端層（SSL），憑證可用來加密網路流量。 這些 TLS/SSL 憑證可以儲存在 Azure 金鑰保存庫中，並允許將憑證安全部署至 Azure 中的 Linux 虛擬機器（VM）。 在本教學課程中，您將了解如何：

> [!div class="checklist"]

> * 設定及保護 Azure 網路
> * 建立 Azure Key Vault
> * 產生或上傳憑證至金鑰保存庫
> * 建立 VM 並安裝 NGINX Web 服務器
> * 將憑證插入 VM，並使用 TLS 系結設定 NGINX

如果您選擇在本機安裝和使用 CLI，本教學課程會要求您執行 Azure CLI 2.0.30 版或更新版本。 執行 `az --version` 以尋找版本。 如果您需要安裝或升級，請參閱 [ 安裝 Azure CLI ]( https://learn.microsoft.com//cli/azure/install-azure-cli ) 。

## 產生憑證並將其儲存在 Azure 金鑰保存庫

Azure 金鑰保存庫保護密碼編譯金鑰和秘密，例如憑證或密碼。 金鑰保存庫可協助簡化憑證管理程式，並讓您能夠控制存取這些憑證的金鑰。 您可以在金鑰保存庫內建立自我簽署憑證，或上傳您已擁有的現有受信任憑證。 在本教學課程中，我們會在金鑰保存庫內建立自我簽署憑證，之後將這些憑證插入執行中的 VM。 此程式可確保部署期間 Web 服務器上已安裝最新的憑證。

下列範例會在所選區域中 * * 建立名為 * $MY_KEY_VAULT * 的 Azure 金鑰保存庫，$REGION保留原則為 7 天。 這表示一旦刪除秘密、金鑰、憑證或金鑰保存庫，它就會在可設定的 7 到 90 個行事曆日期間繼續復原。

```bash
az keyvault create \
    --resource-group $MY_RESOURCE_GROUP_NAME \
    --name $MY_KEY_VAULT \
    --location $REGION \
    --retention-days 7\
    --enabled-for-deployment true -o JSON
```

## 建立 VM

現在使用 az vm create 建立 VM。 使用 --custom-data 參數傳入 cloud-init 組態檔，名為 * cloud-init-nginx.txt * 。
Cloud-init 是一種廣泛使用的方法，可讓您在第一次開機時自訂 Linux VM。 您可以使用 cloud-init 來安裝封裝和寫入檔案，或者設定使用者和安全性。 當 cloud-init 在初始開機程式期間執行時，沒有任何額外的步驟或必要的代理程式可套用您的設定。
當您建立 VM 時，憑證和金鑰會儲存在受保護的 /var/lib/waagent/ 目錄中。 在此範例中，我們會安裝及設定 NGINX 網頁伺服器。

```bash
cat > cloud-init-nginx.txt <<EOF
#cloud-config

# Install, update, and upgrade packages
package_upgrade: true
package_update: true
package_reboot_if_require: true

# Install packages
packages:
  - nginx

write_files:
  - owner: www-data:www-data
  - path: /etc/nginx/sites-available/secure-server
    content: |
      server {
        server_name $FQDN;
        listen 443 ssl http2;
        ssl_certificate /etc/nginx/ssl/${MY_CERT_NAME}.crt;
        ssl_certificate_key /etc/nginx/ssl/${MY_CERT_NAME}.key;
      }
      server {
            listen 80;
            server_name $FQDN;
            return 301 https://$FQDN\$request_uri;
      }

  - owner: root:root
    path: /root/convert_akv_cert.sh
    permissions: "0750"
    content: |
        #!/bin/bash
        runtime="10 minute"; endtime=\$(date -ud "\$runtime" +%s); 
        while [[ \$(date -u +%s) -le \$endtime ]]; do
          if [[ ! -f /etc/nginx/ssl/${MY_KEY_VAULT}.${MY_CERT_NAME} ]]
            then
                sleep 5;
            else
                break;
          fi;
        done
        # Split the file in two (cert and key)
        echo "Creating .key file with private key..."
        openssl rsa -outform pem -in /etc/nginx/ssl/${MY_KEY_VAULT}.${MY_CERT_NAME} -out /etc/nginx/ssl/${MY_CERT_NAME}.key
        echo "Creating .crt file with certificate..."
        openssl x509 -outform pem -in /etc/nginx/ssl/${MY_KEY_VAULT}.${MY_CERT_NAME} -out /etc/nginx/ssl/${MY_CERT_NAME}.crt
runcmd:
  - mkdir /etc/nginx/ssl
  - ln -s /etc/nginx/sites-available/secure-server /etc/nginx/sites-enabled/
  - rm /etc/nginx/sites-enabled/default
  - bash /root/convert_akv_cert.sh
  - (crontab -l 2>/dev/null; echo "0 * * * * /root/convert_akv_cert.sh && service nginx reload") | crontab -
  - service nginx restart
EOF
```
